Section 1

変数と条件分岐の復習

ゲームロジックを書くための基本文法

変数

変数の種類

C#でよく使う4種類の変数です。ゲームのデータを保持するために使います。

using UnityEngine; public class VariableExample : MonoBehaviour { // int：整数（スコア・残機・カウント） int score = 0; int lives = 3; // float：小数（移動速度・タイマー・体力） float speed = 5.0f; float hp = 100.0f; // bool：真偽値（ゲーム中か・ジャンプ中か） bool isGameOver = false; bool isJumping = false; // string：文字列（プレイヤー名・メッセージ） string playerName = "Player"; string message = "Game Start!"; }
if文

if文の基本

条件が成立したときだけ処理を実行します。ゲームのルール判定の基本です。

void Update() { // スコアが100以上ならメッセージを表示 if (score >= 100) { Debug.Log("100点達成！"); } // ゲームオーバー判定 if (hp <= 0) { isGameOver = true; Debug.Log("ゲームオーバー"); } else if (hp <= 30) { Debug.Log("HP残り少ない！"); } else { Debug.Log("まだ余裕あり"); } }
Section 2

スコアシステムを作る

変数・UI・アイテム取得をつなぎ合わせる

Score 変数

スコア変数を作る

スコアを管理するためのシンプルなスクリプトを作成します。

using UnityEngine; public class ScoreSystem : MonoBehaviour { // スコア変数（privateで外部から直接変更させない） private int score = 0; // スコアを加算するメソッド public void AddScore(int amount) { score += amount; Debug.Log("現在のスコア: " + score); } // スコアを取得するメソッド（読み取り専用） public int GetScore() { return score; } // スコアをリセット public void ResetScore() { score = 0; } }
アイテム取得

アイテム取得でスコアを加算する

OnTriggerEnter でアイテムを取得したとき、ScoreSystem を呼び出します。

using UnityEngine; public class CollectibleItem : MonoBehaviour { public int scoreValue = 10; // このアイテムの得点 void OnTriggerEnter(Collider other) { if (other.CompareTag("Player")) { // プレイヤーのScoreSystemを取得してスコア加算 ScoreSystem scoreSystem = other.GetComponent<ScoreSystem>(); if (scoreSystem != null) { scoreSystem.AddScore(scoreValue); } Destroy(gameObject); // アイテムを消す } } }
Score UI

UIにスコアを表示する（TextMeshPro連携）

ScoreSystem を更新するたびに TextMeshProUGUI を書き換えます。

using TMPro; using UnityEngine; public class ScoreSystem : MonoBehaviour { private int score = 0; // InspectorでTextMeshProUGUIを紐づける public TextMeshProUGUI scoreText; public void AddScore(int amount) { score += amount; UpdateScoreUI(); } void UpdateScoreUI() { // スコアをテキストに反映 scoreText.text = "Score: " + score.ToString(); } void Start() { UpdateScoreUI(); // 初期表示 } }
Section 3

GameManagerを作る

ゲーム全体の状態を一元管理するスクリプト

GameManager

GameManagerとは

ゲーム全体の状態（スコア・残機・ゲームオーバーフラグなど）を一か所で管理する専用スクリプトです。

GameManagerが管理するもの

  • スコア・ハイスコア
  • 残機数・ライフ
  • ゲームの状態（開始・プレイ中・ゲームオーバー）
  • シーン遷移の制御

なぜ一元管理するのか

  • スコアが「どこにあるか」を一か所で把握できる
  • 複数のスクリプトから同じデータに安全にアクセスできる
  • デバッグ・修正が楽になる
  • ゲームの規模が大きくなっても管理しやすい

まず空のGameObjectを作成し、名前を「GameManager」にしてからスクリプトをアタッチしましょう。

static

staticを使ったスコア管理

static 変数はどのスクリプトからも GameManager.score でアクセスできます。

using TMPro; using UnityEngine; public class GameManager : MonoBehaviour { // static にすることでどこからでもアクセス可能 public static int score = 0; public static bool isGameOver = false; public TextMeshProUGUI scoreText; // 他スクリプトからスコアを加算するときに呼ぶ public static void AddScore(int amount) { score += amount; } void Update() { // UIを毎フレーム更新（本来はイベント駆動が望ましい） scoreText.text = "Score: " + score; } } // --- 別スクリプトからの呼び出し例 --- // GameManager.AddScore(10); // どこからでも呼べる
Section 4

ミニゲームを作ろう

動くキャラ・アイテム・スコアをつなげて完成させる

ゲーム設計

シンプルなコレクションゲームの設計

今回作るミニゲームの構成要素を整理します。

1
動くキャラクター（Player）Rigidbody + Collider + 移動スクリプト。WASDまたは矢印キーで移動する。
2
アイテム（Collectible）Is Trigger ON のCollider。OnTriggerEnterでスコアを加算して自分を消す。
3
スコア表示（Canvas + TextMeshPro）GameManagerのstatic変数をTextに反映する。
4
ゲームオーバー条件タイマーが0になるか、HPが0になったらisGameOver = true にする。
練習課題

今日の練習課題

アイテムを集めてスコアを表示するゲームを作ろう。

1
プレイヤーを動かすRigidbodyを使いWASDキーで移動できるスクリプトを作成する
2
アイテムを複数配置SphereにIs Trigger ColliderをつけてCollectibleItemスクリプトをアタッチ
3
GameManagerを作成static score変数とAddScore()メソッドをGameManagerに実装する
4
スコアUIを更新アイテム取得のたびにTextMeshProのテキストが更新されることを確認する
まとめ

今日のまとめ

  • int / float / bool / string：ゲームデータを保持する基本的な変数の種類
  • if文：条件に応じて処理を分岐させるゲームロジックの基本
  • AddScore()：スコアを加算しUIに反映する一連の流れ
  • GameManager：ゲーム全体の状態を一元管理する専用スクリプト
  • static変数：どのスクリプトからも GameManager.score でアクセスできる
