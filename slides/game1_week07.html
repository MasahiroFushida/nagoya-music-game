Section 1

物理エンジンの基本

Rigidbody と Collider ― 物理演算の2本柱

Rigidbody

Rigidbodyとは

オブジェクトに物理演算を適用するコンポーネントです。重力・力・速度などを Unity の物理エンジンが自動的に計算します。

主なプロパティ

  • Mass：重さ（kg）
  • Drag：空気抵抗
  • Use Gravity：重力の有無
  • Is Kinematic：物理演算を無効化

使用シーン

  • プレイヤーキャラクター
  • 落下するオブジェクト
  • 物理的な力を受けるもの
  • ジャンプ・移動する物体

Rigidbody を持つオブジェクトは毎フレーム Update ではなく FixedUpdate 内で移動処理を書くのが基本です。

Collider

Colliderとは

オブジェクトの「衝突範囲（当たり判定の形）」を定義するコンポーネントです。

主なColliderの種類

  • Box Collider：直方体形状
  • Sphere Collider：球形状
  • Capsule Collider：カプセル形状（人型キャラに多用）
  • Mesh Collider：メッシュに沿った形状

設定のポイント

  • 視覚的なメッシュより少し小さめにするとゲームが遊びやすい
  • 複雑な形には Mesh Collider より複数の単純 Collider を組み合わせる方が軽い
  • Is Trigger にチェックを入れると物理的な衝突をしなくなる
物理演算

RigidbodyとColliderの関係

A
Rigidbodyのみ形がないので衝突が起きない。重力だけ受ける（実用的でない）
B
Colliderのみ（Static）動かない地面・壁として機能する。物理演算を受けない
C
Rigidbody + Collider（Dynamic）物理演算を受けながら衝突も判定する。通常のプレイヤー・敵に使う

衝突イベント（OnCollisionEnter等）が発生するには、衝突する側の少なくとも一方に Rigidbody が必要です。

Section 2

衝突判定

OnCollisionEnter で物理的な当たりを検知する

OnCollisionEnter

OnCollisionEnterの使い方

2つのオブジェクトが物理的に接触した瞬間に呼ばれるメソッドです。

using UnityEngine; public class PlayerCollision : MonoBehaviour { // 衝突した瞬間に1回だけ呼ばれる void OnCollisionEnter(Collision collision) { Debug.Log("衝突したオブジェクト: " + collision.gameObject.name); } // 衝突している間、毎フレーム呼ばれる void OnCollisionStay(Collision collision) { } // 離れた瞬間に1回だけ呼ばれる void OnCollisionExit(Collision collision) { } }
実例

壁に当たったら止まる処理

Rigidbody の速度をリセットして動きを止めます。

using UnityEngine; public class BallController : MonoBehaviour { private Rigidbody rb; void Start() { rb = GetComponent<Rigidbody>(); } void OnCollisionEnter(Collision collision) { // "Wall" タグのオブジェクトに当たったら速度をゼロにする if (collision.gameObject.CompareTag("Wall")) { rb.linearVelocity = Vector3.zero; Debug.Log("壁に当たった！"); } } }
Section 3

トリガー判定

物理的な壁を作らずに「範囲に入ったこと」を検知する

Is Trigger

Is Triggerの設定方法

ColliderのInspectorで Is Trigger にチェックを入れると、物理的な衝突をせずに「範囲の侵入」だけを検知できます。

1
アイテムオブジェクトを選択ColliderコンポーネントのInspectorを開く
2
Is Trigger にチェックこれでオブジェクトをすり抜けて通過できるようになる
3
スクリプトに OnTriggerEnter を記述プレイヤーが範囲に入ったタイミングで処理を実行する
OnTriggerEnter

OnTriggerEnterの使い方

トリガー範囲に他のオブジェクトが入った瞬間に呼ばれます。

using UnityEngine; public class ItemPickup : MonoBehaviour { // トリガー範囲に入った瞬間に呼ばれる void OnTriggerEnter(Collider other) { Debug.Log("範囲に入ったオブジェクト: " + other.gameObject.name); } // 範囲内にいる間、毎フレーム呼ばれる void OnTriggerStay(Collider other) { } // 範囲から出た瞬間に呼ばれる void OnTriggerExit(Collider other) { } }
引数の型に注意：OnCollisionEnter は Collision、OnTriggerEnter は Collider です。
使い分け

衝突 vs トリガーの使い分け

衝突判定（Collision）

  • 物理的に押しのけあう
  • 壁・床・障害物
  • 跳ね返るボール
  • 積み重なる物体

→ Is Trigger はOFF

トリガー判定（Trigger）

  • すり抜けて検知だけする
  • アイテム取得エリア
  • ゴールゾーン
  • 罠・ダメージ範囲

→ Is Trigger はON

「物理的にぶつかってほしいか？」が判断基準。
ぶつかってほしければ Collision、すり抜けさせたければ Trigger。

Tag

タグ（Tag）を使った判定

Tagを使うと、どのオブジェクトと接触したかを名前ではなくカテゴリで識別できます。

using UnityEngine; public class ItemPickup : MonoBehaviour { void OnTriggerEnter(Collider other) { // "Player" タグのオブジェクトが触れた場合のみ処理する if (other.CompareTag("Player")) { Debug.Log("プレイヤーがアイテムを取得！"); // このオブジェクト（アイテム）を消す Destroy(gameObject); } } }
Tagは事前に Inspector の Tag ドロップダウン → Add Tag から追加し、オブジェクトに設定しておく必要があります。
練習課題

今日の練習課題

アイテムに触れると消える処理を実装しよう。

1
プレイヤーを作成CapsuleにRigidbody + Capsule Colliderをつけ、Tagを「Player」に設定
2
アイテムを作成SphereにSphere Colliderをつけ、Is TriggerをONにする
3
ItemPickupスクリプトを作成OnTriggerEnterで「Player」タグを確認し、Destroy(gameObject)で消す
4
アイテムを複数配置して確認プレイヤーが触れたアイテムだけが消えることをテストする
まとめ

今日のまとめ

  • Rigidbody：物理演算（重力・速度など）を有効にするコンポーネント
  • Collider：当たり判定の形を定義するコンポーネント
  • OnCollisionEnter：物理的な衝突の瞬間に呼ばれる（引数：Collision）
  • Is Trigger + OnTriggerEnter：すり抜けて範囲侵入を検知する（引数：Collider）
  • CompareTag()：Tagでオブジェクトを識別する安全な方法
