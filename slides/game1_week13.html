動作確認

動作確認のポイント

  • こまめに Play モードで確認する― 「5分に1回」ではなく「1機能追加したら即テスト」を習慣に
  • コンパイルエラーが出たらすぐ止まる― エラーを積み重ねるとデバッグが困難になる
  • タイトル→ゲーム→クリア→リトライの流れを通して確認― 部分だけでなく全体の流れをテストする
  • 別の人に1回遊んでもらう― 自分では気づかない UI の分かりにくさが発見できる

Play モードで動作確認 → エラー発見 → 修正 → 再確認
このサイクルを素早く回すことがゲーム開発の基本です。

演出調整

UI と音響演出の調整

UI のバランス調整

  • フォントサイズは大きすぎず小さすぎず（最低 24pt）
  • スコア・タイマーは画面の端（角）に配置
  • 重要な情報（残機・HP）は目立つ色で
  • Canvas の Render Mode は "Screen Space - Overlay" が基本

音響のバランス調整

  • BGM は -10〜-20 dB に抑えて SE と被らないようにする
  • SE は短く（0.5秒以内）が聴きやすい
  • クリア音は明るく・失敗音は低く
  • AudioMixer でマスターボリュームを管理する

音響設計は音楽大学生の最大の強みです。
BGM の BPM や調性をゲームの雰囲気に合わせることで作品の質が大きく上がります。

トラブル対処

困ったときの対処法

1
エラーメッセージを読むConsole のエラーをクリックするとエラーが発生したファイルと行番号がわかる。まずここを確認する
2
エラーメッセージで検索する"NullReferenceException Unity" のように Unity を付けて検索すると公式 Docs や解決例が見つかる
3
Debug.Log で値を確認する変数が想定外の値になっていることが多い。怪しい箇所に Debug.Log を挿入して値を確認する
4
先生・隣の人に質問する15分以上詰まったら質問する。一人で抱え込まず声をかけてください
