Section 1

タイマーを実装する

Time クラスの仕組みを理解して
カウントダウン処理を書いてみよう

Time クラス

Time.time と Time.deltaTime

Time.time

  • ゲーム開始からの
    経過秒数（累積）
  • 常に増加し続ける
  • イベント発生時刻の
    記録などに使う

Time.deltaTime

  • 直前のフレームから
    今フレームまでの秒数
  • フレームレートに
    依存しない処理に必須
  • タイマー減算に使う

ポイント：タイマーを減らすときは必ず Time.deltaTime を掛ける。
こうすることで PC の性能に関係なく同じ速度でカウントされる。

実装

カウントダウンタイマーの実装

using UnityEngine; public class GameTimer : MonoBehaviour { public float timeLimit = 30f; // 制限時間（秒） private float timeRemaining; private bool isRunning = true; void Start() { timeRemaining = timeLimit; } void Update() { if (!isRunning) return; timeRemaining -= Time.deltaTime; // 毎フレーム減算 if (timeRemaining <= 0f) { timeRemaining = 0f; isRunning = false; OnTimeUp(); // 時間切れ処理を呼ぶ } } void OnTimeUp() { Debug.Log("時間切れ！"); } }
UI

UI にタイマーを表示する

TextMeshProUGUI を使って残り時間をリアルタイム表示する

using UnityEngine; using TMPro; public class GameTimer : MonoBehaviour { public float timeLimit = 30f; public TextMeshProUGUI timerText; // Inspector で割り当て private float timeRemaining; private bool isRunning = true; void Start() { timeRemaining = timeLimit; } void Update() { if (!isRunning) return; timeRemaining -= Time.deltaTime; if (timeRemaining <= 0f) { timeRemaining = 0f; isRunning = false; } // "00:29" 形式で表示 timerText.text = Mathf.CeilToInt(timeRemaining).ToString("00"); } }
Section 2

クリア・失敗条件

「どうなったら勝ちか？負けか？」を
コードで明確に定義しよう

ゲームクリア

ゲームクリア条件の例

  • フィールド内の全アイテムを取得した
  • 制限時間内にゴール地点に到達した
  • スコアが目標値を超えた
  • 敵を全滅させた
  • 指定の音楽フレーズに合わせて全ノーツを叩いた（リズムゲーム系）

クリア条件は1つに絞ると設計がシンプルになります。
最初は「アイテムを全部取ったらクリア」などから始めましょう。

ゲームオーバー

ゲームオーバー条件の例

  • プレイヤーの体力（HP）が 0 になった
  • 制限時間が 0 になった（時間切れ）
  • 落下してステージ外に出た
  • 敵に触れた回数が上限を超えた
クリア条件とゲームオーバー条件が重複しないよう注意！
例：「時間切れ→クリア」と「時間切れ→ゲームオーバー」は同時に設定しない。
コード例

条件判定のコード例

public class GameManager : MonoBehaviour { public int totalItems = 5; // 取得すべきアイテム数 private int collectedItems = 0; private bool gameEnded = false; // アイテム取得時に呼び出す public void OnItemCollected() { collectedItems++; if (collectedItems >= totalItems) { GameClear(); } } // HP または時間切れ時に呼び出す public void OnGameOver() { if (gameEnded) return; gameEnded = true; Debug.Log("ゲームオーバー"); // → ゲームオーバーシーンへ遷移 } void GameClear() { if (gameEnded) return; gameEnded = true; Debug.Log("ゲームクリア！"); // → クリアシーンへ遷移 } }
Section 3

シーン遷移

タイトル → ゲーム → クリア/オーバー
Scene を切り替えてゲームの流れを作ろう

SceneManager

SceneManager とは

  • Unity でシーン（画面）を管理するクラス
  • UnityEngine.SceneManagement 名前空間に含まれる
  • シーンの読み込み・切り替え・追加ロードができる

よく使うメソッド

  • LoadScene(名前 or 番号)
  • LoadSceneAsync(…)
  • GetActiveScene()

シーンの構成例

  • 0: TitleScene
  • 1: GameScene
  • 2: ClearScene
  • 3: GameOverScene
Build Settings

シーンを追加する手順（Build Settings）

1
新しいシーンを作成File > New Scene → 保存（Ctrl+S）で .unity ファイルを作る
2
Build Settings を開くFile > Build Settings（Ctrl+Shift+B）
3
シーンを追加"Add Open Scenes" ボタンをクリック、または Scenes フォルダからドラッグ＆ドロップ
4
順番を確認Index 0 が起動時に最初に読み込まれるシーン（タイトル画面）になる
コード

シーン遷移のコード

using UnityEngine; using UnityEngine.SceneManagement; // 必ず using を追加 public class SceneController : MonoBehaviour { // シーン名で遷移 public void GoToGame() { SceneManager.LoadScene("GameScene"); } // Build Index（番号）で遷移 public void GoToGameOver() { SceneManager.LoadScene(3); // index 3 = GameOverScene } // 現在のシーンを再読み込み（リトライ） public void Retry() { Scene current = SceneManager.GetActiveScene(); SceneManager.LoadScene(current.name); } }
Build Settings にシーンを追加しておかないと実行時にエラーになります。
