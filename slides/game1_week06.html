Section 1

UIとは

User Interface ― プレイヤーとゲームをつなぐ情報窓口

UI とは

UI（ユーザーインターフェース）

プレイヤーがゲームの状態を把握し、操作するための視覚的な要素の総称です。

表示系UI

  • スコア表示
  • 体力バー（HP）
  • 残り時間・残機数
  • ミニマップ

操作系UI

  • スタートボタン
  • ポーズメニュー
  • 設定画面
  • ゲームオーバー画面

UIはゲームの「伝える力」。情報が伝わらないと、どんなに面白いゲームでもプレイヤーが混乱します。

Canvas

CanvasはUIの土台

UnityではすべてのUI要素は Canvas の子オブジェクトとして配置します。

1
Canvasオブジェクト画面全体を覆う「透明な板」。UI要素を描画する専用レイヤー。
2
EventSystemCanvasと同時に自動生成される。ボタンのクリックなどのイベントを管理する。
3
Render Mode通常は「Screen Space - Overlay」を使用。常に画面の最前面に表示される。
Section 2

Text・Image・Button
を作ろう

UnityのHierarchyからUI要素を追加する

Text

TextMeshProの配置

通常のTextよりも高品質なフォント描画が可能な TextMeshPro を使います。

1
Hierarchyを右クリックコンテキストメニューを開く
2
UI → Text - TextMeshProを選択
3
TMP Importerが出たら「Import TMP Essentials」をクリック
4
Canvasの子に自動配置されることを確認する
TextMeshPro 設定

TextMeshProの主要設定

Inspector で設定できる項目

  • Text Input：表示するテキスト内容
  • Font Asset：使用するフォント
  • Font Size：文字の大きさ
  • Vertex Color：文字の色
  • Alignment：文字揃え（左・中・右）

Rect Transform で位置調整

  • Anchor：基準点を設定
  • Pos X / Y：位置を指定
  • Width / Height：テキスト領域のサイズ
  • Anchor Presetsで画面端への固定が簡単にできる
日本語を表示するにはフォントアセットに日本語対応フォントを設定する必要があります。
Image

Imageの配置と設定

UIで画像を表示するには Image コンポーネントを使います。

1
Hierarchy右クリック → UI → ImageImageオブジェクトが作成される
2
Source Image使いたい画像（Sprite）をInspectorにドラッグ＆ドロップ
3
Image Type「Filled」にすると体力バーのような減少アニメーションが作れる

テクスチャをSpriteとしてImportする際は、Inspector の Texture Type を「Sprite (2D and UI)」に変更してください。

Button

Buttonの配置

ボタンはクリックやタップに反応するインタラクティブなUI要素です。

1
Hierarchy右クリック → UI → Button - TextMeshProButton と子のTextが一緒に作成される
2
子のTextを選択InspectorでボタンのラベルテキストをTextInputで変更する
3
Button本体のInspector「Transition」でホバー・クリック時の色変化を設定できる
Section 3

ボタンにイベントをつける

クリックしたときに何かが起こる仕組みを作る

OnClick

OnClickイベントの設定

ButtonコンポーネントのInspectorに On Click () セクションがあります。

1
On Click () の「+」をクリックイベントの受け取り先を追加する
2
オブジェクトをドラッグ左のフィールドに呼び出したいスクリプトを持つオブジェクトをセット
3
メソッドを選択右のプルダウンから呼び出すメソッド名を選ぶ

スクリプト側では public なメソッドを定義しておく必要があります。private では一覧に表示されません。

Script × UI

スクリプトからUIを操作する

TextMeshProUGUI コンポーネントを取得してテキストを書き換えます。

using TMPro; using UnityEngine; public class UIController : MonoBehaviour { // InspectorでTextMeshProUGUIを紐づける public TextMeshProUGUI scoreText; void Start() { scoreText.text = "Score: 0"; } public void UpdateScore(int score) { scoreText.text = "Score: " + score; } }
using TMPro; を忘れずに記述してください。TextMeshProUGUI は TMPro 名前空間に含まれています。
Section 4

スコア表示を作ろう

変数の値をリアルタイムにUIへ反映する

Score UI

スコア変数をUIに反映する

アイテムを取得するたびにスコアを加算し、表示を更新します。

using TMPro; using UnityEngine; public class ScoreManager : MonoBehaviour { public TextMeshProUGUI scoreText; private int score = 0; void Start() { UpdateUI(); } public void AddScore(int amount) { score += amount; UpdateUI(); } void UpdateUI() { scoreText.text = "Score: " + score.ToString(); } }
まとめ

今日のまとめ

  • UIはプレイヤーとゲームをつなぐ情報要素（スコア・体力・ボタンなど）
  • すべてのUI要素は Canvas の子として配置する
  • テキストは TextMeshPro、画像は Image、操作は Button を使う
  • ボタンのクリック処理は OnClick にメソッドを登録する
  • スクリプトから TextMeshProUGUI.text を書き換えてUIを更新する
