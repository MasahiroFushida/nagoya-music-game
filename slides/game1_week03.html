Section 01

スクリプトとは何か

オブジェクトに「頭脳」を与えるしくみ

スクリプトの役割

オブジェクトに「動き」と「ルール」を与える

  • Unityのオブジェクトは初期状態では「置いてあるだけ」で何もしない
  • スクリプトを使うことで「キーを押したら動く」「敵と当たったら減点」などを実現できる
  • スクリプトはコンポーネントの一種 ― Inspector から追加・削除できる
  • 1つのオブジェクトに複数のスクリプトを付けることも可能

スクリプト ＝ オブジェクトへの「指示書き」。書いた内容がゲームの動作に直結します。

C# とは

Unityが使うプログラミング言語

  • 読み方：「シーシャープ」（C-Sharp）
  • Microsoft が開発した汎用プログラミング言語で、Unityの公式スクリプト言語
  • 「.cs」という拡張子のファイルに書く（例：PlayerController.cs）
  • 文法は英語ベース。大文字・小文字が区別される
  • セミコロン（;）で文の終わりを表す

C# でできること

オブジェクトの移動・回転、当たり判定、スコア管理、UI更新、音の再生…

Section 02

スクリプトを作ってみよう

作成 → 編集 → アタッチの流れ

スクリプト作成

C# スクリプトの作り方

1
Project ウィンドウを右クリックAssets フォルダ（または Scripts フォルダ）の空白エリアで右クリック
2
Create → C# Scriptメニューから選択する
3
ファイル名を入力してEnter例：「HelloScript」。作成直後に名前を変更すること！（後から変えると不整合が起きる）
4
ダブルクリックで Visual Studio（またはRider）を開くコードエディターが起動してテンプレートが表示される
スクリプトの基本構造

Start と Update

using UnityEngine; public class HelloScript : MonoBehaviour { // ゲーム開始時に1回だけ呼ばれる void Start() { // ここに初期化処理を書く } // 毎フレーム呼ばれる（約60回/秒） void Update() { // ここに毎フレームの処理を書く } }

Start：ゲーム開始時に1度だけ実行。初期設定に使う。
Update：毎フレーム（毎秒約60回）実行。動きやチェックに使う。

Debug.Log

コンソールに文字を出力する

void Start() { Debug.Log("Hello, Unity!"); Debug.Log("自分の名前: 山田 太郎"); }
  • Debug.Log( ) の括弧内に文字列（"…"で囲む）や変数を書く
  • 実行すると Unity の Console ウィンドウに出力される
  • エラーではなく「開発者向けのメモ」として使う機能
  • 完成したゲームには影響しない（パフォーマンス上は多用を避ける）
アタッチ

スクリプトをオブジェクトに付ける

1
Hierarchy でオブジェクトを選択例：Cube をクリックして選択状態にする
2
Project の .cs ファイルを Inspector にドラッグまたはInspector の「Add Component」→ スクリプト名を検索
3
Inspector にスクリプトが表示されれば成功コンポーネントとして追加されたことを確認する
⚠ スクリプト名とクラス名（class の後ろ）が一致していないとアタッチできません。
Section 03

Playモードで確認しよう

実行・コンソール・よくあるエラー

Playモード

ゲームを実行して確認する

  • エディター上部の ▶（Play）ボタンをクリックでPlayモード開始
  • もう一度クリックで停止。Pause（⏸）で一時停止も可能
  • Playモード中は Game ビューがアクティブになる
⚠ Playモード中に変更した内容は保存されません！
Stop してから編集するのが基本。必ずPlayモードを終了してから変更・保存しましょう。

エディターが青っぽく光っている間は Playモード中のサインです。

Console ウィンドウ

Debug.Log の出力を確認する

  • メニュー Window → General → Console で開く（またはエラー発生時に自動で開く）
  • Debug.Log の出力は白いテキストで表示される
  • 黄色い警告（Warning）：問題になりそうな状況を知らせる
  • 赤いエラー（Error）：コードに問題があり、ゲームが止まる可能性がある
  • ログをクリックすると、どのスクリプトの何行目かが表示される

Console はデバッグの最強の味方。エラーが出たらまずここを読む習慣を！

よくあるエラー

スペルミス・大文字小文字に注意

よくある間違い例

  • debug.log → Debug.Log（大文字！）
  • Void → void（小文字！）
  • セミコロン忘れ（文末の ;）
  • 括弧の対応ミス { } ( )

対処法

  • Console のエラーメッセージを読む
  • 何行目(line XX)かを確認する
  • Visual Studio の赤い下線を確認
  • コピー＆ペーストを活用する
⚠ C# は大文字・小文字が完全に区別されます。Debug.Log と debug.log は別物です。
練習課題

Debug.Log で自分の名前を出力しよう

1
C# スクリプトを新規作成する名前は「MyFirstScript」にする
2
Start() の中に Debug.Log を書く例：Debug.Log("私の名前は〇〇です！");
3
Cube にアタッチして Playモードで確認Console に自分の名前が表示されれば完成！

できた人は、Update() の中にも書いて Console がどう変わるか観察してみよう。

まとめ

今日学んだこと

  • スクリプトはオブジェクトに動きとルールを与えるコンポーネント
  • Unity のスクリプト言語は C#（シーシャープ）
  • Project → Create → C# Script で作成し、ダブルクリックで編集
  • Start()：開始時に1回だけ実行 / Update()：毎フレーム実行
  • Debug.Log("…") でコンソールに出力して動作を確認できる
  • Playモード中の変更は保存されない！必ず Stop してから編集する
