Section 1

ゲームにおける
音の役割

音楽大学生の視点でゲームサウンドを考える

ゲームサウンド

SE・BGM・ジングルの違いと役割

SE
サウンドエフェクト（効果音）ジャンプ音、コイン取得音、ダメージ音など。プレイヤーの行動や状態変化を即座にフィードバックする短い音。
BGM
バックグラウンドミュージック（背景音楽）ゲームプレイ中にループ再生される楽曲。場面の雰囲気・テンション・世界観を作る。
JG
ジングル（ステートミュージック）ゲームクリア・ゲームオーバーなど特定の状態遷移で1回だけ鳴らす短い楽曲。

音はゲームの「体験」を作る重要な要素。音がないゲームは感情的な手がかりを失います。

音声フォーマット

Unityで使える音声フォーマット

.wav（非圧縮）

ファイルサイズが大きいが高音質。短い効果音に最適。メモリに展開して即時再生できるため、SEに推奨。

.mp3（圧縮）

ファイルサイズが小さい。BGMなど長い楽曲に向いている。ただしループポイントがずれる場合がある。

.ogg（Vorbis圧縮）

ゲーム開発での標準的な圧縮フォーマット。音質と圧縮率のバランスが良く、ループも正確。BGMにはmpよりoggを推奨。

Unityでは Import Settings の Load Type・Compression Format でメモリ使用量と品質を調整できます。
Section 2

AudioSourceを使おう

音を鳴らすためのコンポーネントを設定する

AudioSource

AudioSourceコンポーネントの追加

AudioSource は「スピーカー」の役割を果たすコンポーネントです。音を鳴らしたいオブジェクトにアタッチします。

1
オブジェクトを選択音を鳴らしたいGameObjectをHierarchyで選ぶ
2
Add ComponentInspectorのAdd Component → Audio → Audio Source を選択
3
AudioClipを設定プロジェクトの音声ファイルをAudio Clipフィールドにドラッグ&ドロップ
AudioSource 設定

AudioSourceの主要設定

基本設定

  • Audio Clip：再生する音声ファイル
  • Volume：音量（0〜1）
  • Pitch：ピッチ（1 = 標準）
  • Loop：ループ再生するか

再生タイミング

  • Play On Awake：ゲーム開始時に自動再生
  • Priority：再生優先度（0=高〜256=低）
  • Spatial Blend：0=2D音源、1=3D空間音響
  • BGMは通常 2D（Spatial Blend = 0）にする
BGM設定

BGMのループ再生設定

空のGameObjectを作成してBGM専用のAudioSourceをアタッチするのが一般的です。

1
空のGameObjectを作成Hierarchy右クリック → Create Empty → 名前を「BGMManager」等に変更
2
AudioSourceを追加BGM用音声ファイルをAudio Clipに設定する
3
LoopをON、Play On AwakeをONこれだけでゲーム開始と同時にBGMがループ再生される

Spatial Blend を 0（2D）に設定しないと、カメラから離れると音量が下がってしまいます。BGMは必ず 0 にしてください。

Section 3

スクリプトから
音を再生する

Play() と PlayOneShot() を使い分ける

Play / PlayOneShot

AudioSource.Play() / PlayOneShot()

using UnityEngine; public class SoundTest : MonoBehaviour { private AudioSource audioSource; public AudioClip seClip; // Inspector でSEをセット void Start() { audioSource = GetComponent<AudioSource>(); } void Update() { // Spaceキーで再生（途中で呼ぶと最初からになる） if (Input.GetKeyDown(KeyCode.Space)) { audioSource.Play(); } // Enterキーで重ね再生（同時に複数鳴らせる） if (Input.GetKeyDown(KeyCode.Return)) { audioSource.PlayOneShot(seClip); } } }
SE + Trigger

効果音を接触時に鳴らす

OnTriggerEnter と組み合わせてアイテム取得SEを実装します。

using UnityEngine; public class ItemWithSound : MonoBehaviour { public AudioClip pickupSound; // Inspector でSEをセット private AudioSource audioSource; void Start() { audioSource = GetComponent<AudioSource>(); } void OnTriggerEnter(Collider other) { if (other.CompareTag("Player")) { // 音を鳴らしてからオブジェクトを消す AudioSource.PlayClipAtPoint(pickupSound, transform.position); Destroy(gameObject); } } }
Destroy する前に audioSource.Play() を呼んでも、オブジェクトと一緒に AudioSource も消えます。PlayClipAtPoint は一時的なオブジェクトを自動生成して再生するので安全です。
Section 4

音量・ピッチの制御

スクリプトから動的にサウンドパラメータを変える

Volume / Pitch

音量・ピッチをスクリプトから変える

using UnityEngine; public class AudioController : MonoBehaviour { private AudioSource audioSource; void Start() { audioSource = GetComponent<AudioSource>(); } // 音量を変える（0.0 〜 1.0） public void SetVolume(float volume) { audioSource.volume = Mathf.Clamp01(volume); } // ピッチを変える（0.5 = 半音下、2.0 = 1オクターブ上） public void SetPitch(float pitch) { audioSource.pitch = pitch; } // 一時停止と再開 public void TogglePause() { if (audioSource.isPlaying) audioSource.Pause(); else audioSource.UnPause(); } }
音楽×ゲーム

音楽大学生へのヒント

音楽の知識はゲームサウンドデザインの大きな強みになります。

1
ピッチでバリエーションを出す同じSEでも pitch をランダム（0.9〜1.1）にするだけで繰り返しの単調さがなくなる
2
音量でフィードバックを伝える強い攻撃は大きく、遠くの音は小さく。音量差でゲームの情報を伝える
3
インタラクティブミュージックゲームの状態（緊張・解放）に合わせてBGMのテンポや音量を動的に変化させると体験が深まる

自分で作った音源をUnityに入れてゲームと組み合わせることができます。ぜひオリジナルサウンドに挑戦してみてください。

練習課題

今日の練習課題

ジャンプ音・BGMをゲームに設定しよう。

1
BGMを設定する空オブジェクトにAudioSourceを追加。Loop ON・Play On Awake ONにしてBGMをセットする
2
ジャンプSEを追加プレイヤースクリプトのジャンプ処理にPlayOneShot()を追加する
3
アイテム取得SEを追加PlayClipAtPoint()を使ってアイテムが消える瞬間に音を鳴らす
4
チャレンジ：ランダムピッチSEのpitchをRandom.Range(0.9f, 1.1f)にして毎回微妙に変える
まとめ

今日のまとめ

  • SE / BGM / ジングル：それぞれ役割が異なる。用途に合わせて使い分ける
  • AudioSource：音を鳴らすコンポーネント。AudioClipをセットして再生する
  • Play()：AudioSourceのClipを再生（重ね再生不可）
  • PlayOneShot(clip)：指定Clipを重ね再生できる。SEに最適
  • PlayClipAtPoint()：オブジェクトが消えてもその場所で音だけ鳴らす
  • volume / pitch：プロパティを代入するだけで動的に変更できる
