Section 1

音楽とゲーム演出

音楽大学生ならではの視点で
ゲームの「音」を考えてみよう

音楽大学生の視点

インタラクティブ音楽とは

ゲームの状態に応じて音楽が変化する仕組みのこと。映画やコンサートの「一方向」な音楽と異なり、プレイヤーの行動が音楽を変える。

状態連動の例

  • 戦闘中 → テンポが上がる
  • HP が低い → 音楽が不安定に
  • クリア時 → ファンファーレ
  • 水中 → 音がこもった感じに

Unity での実現方法

  • AudioSource の pitch を変える
  • 複数 BGM を CrossFade する
  • AudioMixer でエフェクトを切替
  • Wwise・FMOD 等の外部ツール

まずは「効果音を鳴らす」→「BGM を流す」→「状態で切り替える」の順で実装していきましょう。

Animation

アニメーションとは（Unity のシステム）

  • Animation Clip ― オブジェクトの動きを記録したデータ（.anim ファイル）
  • Animator Controller ― クリップを状態機械（ステートマシン）で管理するグラフ
  • Animator コンポーネント ― GameObject に付けてコントローラーを駆動する

音楽と同様に「状態遷移」の考え方が重要です。
"待機 → 走る → ジャンプ" はまるで楽曲の「A メロ → サビ → アウトロ」のような構造です。

Section 2

Animator を使う

キャラクターに「待機」「走る」などの
アニメーションをつけてみよう

Animator Controller

Animator Controller の作り方

1
Animator Controller を作成Project ウィンドウで右クリック → Create > Animator Controller
2
Animation Clip をドロップAnimator ウィンドウを開き、クリップをドラッグ＆ドロップ → ステートが自動生成される
3
トランジションを設定ステートを右クリック → "Make Transition" で矢印を引き、Conditions（条件）を設定する
4
GameObject に割り当てAnimator コンポーネントの "Controller" 欄に作った Controller をセットする
Animation Clip

アニメーションクリップの作成

1
Animation ウィンドウを開くWindow > Animation > Animation（Ctrl+6）
2
オブジェクトを選択して録画開始対象の GameObject を選択し、"Create" → クリップを保存 → ● 録画ボタンをオン
3
キーフレームを打つタイムラインを移動しながら Transform や色などを変更すると自動でキーフレームが追加される
4
録画終了 → 再生確認● ボタンをオフ → Play ボタンでプレビュー
コード

スクリプトからアニメーションを再生する

using UnityEngine; public class PlayerAnimator : MonoBehaviour { private Animator anim; void Start() { anim = GetComponent<Animator>(); } void Update() { // 移動速度に応じてブレンドツリーを切替 float speed = GetComponent<Rigidbody>().velocity.magnitude; anim.SetFloat("Speed", speed); // ジャンプボタンでトリガーを発火 if (Input.GetButtonDown("Jump")) { anim.SetTrigger("Jump"); } } // アイテム取得時にアニメーション再生 public void PlayPickup() { anim.SetTrigger("Pickup"); } }
Section 3

タイミングに合わせた演出

Invoke と Coroutine で
時間差・連続処理を実装しよう

Invoke

Invoke・InvokeRepeating で時間差処理

public class TimingExample : MonoBehaviour { void Start() { // 2秒後に ShowEffect を1回呼ぶ Invoke("ShowEffect", 2.0f); // 1秒後から0.5秒ごとに繰り返し呼ぶ InvokeRepeating("SpawnEnemy", 1.0f, 0.5f); } void ShowEffect() { Debug.Log("エフェクト発動！"); } void SpawnEnemy() { Debug.Log("敵をスポーン"); } void OnDestroy() { CancelInvoke(); // すべての Invoke をキャンセル } }

シンプルな遅延処理は Invoke、複雑な流れは次のスライドの Coroutine が便利。

Coroutine

Coroutine（コルーチン）の基本

using UnityEngine; using System.Collections; public class CoroutineExample : MonoBehaviour { void Start() { StartCoroutine(PlaySequence()); } IEnumerator PlaySequence() { Debug.Log("1. ゲームスタート！"); yield return new WaitForSeconds(1.0f); // 1秒待つ Debug.Log("2. 敵が出現！"); yield return new WaitForSeconds(2.0f); // 2秒待つ Debug.Log("3. BGM がクライマックスへ"); yield return new WaitForSeconds(0.5f); Debug.Log("4. エフェクト発動！"); } }

yield return で処理を一時停止できる。演出シーケンスの記述にとても適している。

音楽大学生の視点

音楽のビートに合わせたオブジェクト演出

BPM（テンポ）から1拍の長さ（秒）を計算して Coroutine で同期させる方法

public class BeatSync : MonoBehaviour { public float bpm = 120f; void Start() { float beatInterval = 60f / bpm; // 1拍 = 0.5秒（120BPM の場合） InvokeRepeating("OnBeat", 0f, beatInterval); } void OnBeat() { // 1拍ごとにオブジェクトをスケールアップ transform.localScale = Vector3.one * 1.2f; Invoke("ResetScale", 0.1f); } void ResetScale() { transform.localScale = Vector3.one; } }

曲のBPMを正確に設定するだけで、ビジュアルが音楽に完全同期します。
作曲知識がある皆さんにとって、これは大きなアドバンテージです！

ParticleSystem

ParticleSystem で視覚エフェクト

1
Particle System を追加Hierarchy で右クリック → Effects > Particle System
2
主なパラメータを調整Duration（持続時間）・Start Speed（初速）・Start Color（色）・Emission Rate（発生数/秒）
3
スクリプトから制御GetComponent<ParticleSystem>().Play() で再生、.Stop() で停止

「音に合わせてパーティクルを爆発させる」演出はクリア感・達成感を大幅に高めます。
色と動きは楽曲のムードに合わせて設定してみましょう。
