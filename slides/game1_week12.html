Section 2

企画を考えよう

「どんなゲームを作るか」を決めることが
制作全体の方向性を決める最重要ステップ

企画のコツ

良いゲーム企画の条件

  • 小さくてもいいので完成させる― 未完成の大作より完成した小品の方が高評価
  • ルールが1文で説明できる― 「〇〇を避けながら□□を取る」など
  • 操作が直感的― キーボード 1〜2 キーで遊べる範囲に収める
  • 勝ち負けが明確― クリア条件・ゲームオーバー条件をはっきり決める
  • 音楽・音響を活かせる設計― 自分の強みを発揮できる場面を作る
「最初の企画の半分の規模」でちょうど良い完成度になることが多いです。
欲張りすぎず、まずコアとなるゲームループを完成させましょう。
テーマ例

音楽大学生ならではのテーマ例

リズム・音楽系

  • 落ちてくるノーツを叩く
    シンプルリズムゲーム
  • BPM に合わせてジャンプ
    するプラットフォーマー
  • 音楽に合わせて色が変わる
    障害物避けゲーム

演出・世界観系

  • ピアノの鍵盤を踏んで
    進む探索ゲーム
  • 楽器のパーツを集めて
    曲を完成させるパズル
  • 自作 BGM が主役の
    シューティング・アクション

自分で作曲した音楽をゲーム BGM に使うのは最高のアピールポイントになります！
Freesound・Pixabay の素材でも十分ですが、自作音楽を使えれば加点対象です。

企画書

企画書の項目

A4 1 枚（手書き可）または Google ドキュメントでまとめること

  • タイトル― 作品の名前
  • ジャンル― アクション / パズル / リズム / シューティング 等
  • ゲームの説明― 1〜2 文でどんなゲームか説明する
  • 操作方法― キー配置・マウス操作
  • クリア条件・ゲームオーバー条件
  • 使用する音楽・SE― 自作 or 素材サイト名・ファイル名
  • 目標スコープ― 第14回終了時点で何が完成しているか
Section 3

素材を整理しよう

必要な素材をリストアップし
著作権に注意して収集しよう

素材リスト

必要な素材リスト

ビジュアル素材

  • 3D モデル（.fbx / .obj）
    または 2D スプライト（.png）
  • 背景・タイルマップ
  • UI アイコン・ボタン画像
  • フォント（日本語対応）
  • ParticleSystem 用テクスチャ

オーディオ素材

  • BGM（.mp3 / .ogg / .wav）
  • SE：ジャンプ・取得・クリア・
    ゲームオーバー等
  • 環境音・アンビエンス
  • 自作曲があればぜひ！

Unity の Project ウィンドウで Assets / Audio / BGM・SE のようにフォルダを整理しておくと後の作業がスムーズです。

素材サイト

フリー素材サイトの紹介

ビジュアル・3D

  • Unity Asset Store
    Free フィルターで検索

オーディオ

  • Freesound.org
    SE・環境音・CC ライセンス
  • Pixabay（Music）
    BGM 無料・商用可
  • 魔王魂
    日本語サイト・ゲーム用 SE 多数
  • 効果音ラボ
    シンプルな SE が充実
素材を使う場合はライセンス（CC0・CC-BY 等）を必ず確認し、
企画書に出典を記録しておいてください。
